# Template: Mission

> [!WARNING] Mission is **user territory**
> The agent may *propose* a mission, but it may *never establish* one without explicit user approval.
> This template is intended to be drafted by the agent, then opened in Obsidian by the user for inline editing.

A mission is the **north star** — what you and your user are building toward together. It should be ambitious but grounded, specific about domains, and honest about limitations.
